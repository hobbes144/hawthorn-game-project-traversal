25/*!****************************************************************************
26 * \brief Class Renderer that implements an OpenGL renderer
27 *
28 * ## Usage:
29 *
30 * Renderer is needed to render objects onto the screen. This needs to be
31 * initialized to work with a window, and the draw call must be made to draw
32 * objects to the screen.
33 *
34 * ## Pre-initialization calls:
35 *
36 * - setGameWindow(GameWindow* _gameWindow)
37 *
38 * ## Post-initialization calls:
39 *
40 * - setIs3D(bool is3D)
41 * - setClearColor(float r, float g, float b, float a)
42 *
43 * ## General lifecycle of a Renderer:
44 *
45 * - Init GameWindow object.
46 * - Init Renderer.
47 * - Call clear() to empty the window buffer.
48 * - Run all your game logic.
49 * - Run draw() calls for each of the objects to be rendered.
50 * - Call swapBuffers() to render the window buffer to the screen.
51 *
52 * ## Notes:
53 *
54 * Renderer class is specific to a window. It is possible to have multiple
55 * renderers, one for each window you have.
56 *
57 *****************************************************************************/
