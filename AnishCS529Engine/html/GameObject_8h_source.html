23/*!****************************************************************************
24 * \brief GameObject class that implements a single object in the scene
25 *
26 * ## Usage:
27 *
28 * GameObjects are the main objects in the game that represent anything from
29 * characters to terrain to hidden triggers and so on. GameObjects are meant to
30 * be handled by the SceneGraph, and will be called from the
31 * SceneGraph::update(float deltaTime) call in the order they exist in the node
32 * tree.
33 *
34 * ## Pre-initialization calls:
35 *
36 * - addComponent(std::shared_ptr<Component> component)
37 *
38 * ## General lifecycle of a GameObject:
39 *
40 * - Call addComponent() to add the required components. The update function
41 * will return the relevant Component once created.
42 * - Perform pre-initialization calls for each Component added.
43 * - Call update() in the main loop.
44 * - If GameObject is no longer needed, call disable() to disable the object.
45 * - If garbage collection system is implemented, call markForDeletion() and
46 * handle registration to the garbage collection as needed.
47 * - Call shutdown at the end of the main loop or in garbage collection system.
48 *
49 *****************************************************************************/
